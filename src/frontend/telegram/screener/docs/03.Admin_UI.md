Based on your requirements and best practices from modern Telegram bot admin panels, here is a **feature list** and **UI mockups description** for your admin panel. This combines insights from real-world implementations and design resources[1][2][3][4].

## Admin Panel Feature List

### 1. **Authentication & Security**
- **Admin login** (username/password or Telegram OAuth)
- **Role-based access** (e.g., Superadmin, Admin, Support)
- **Session management** and secure logout

### 2. **User Management**
- **View all users** (search, filter, sort by registration date, email, verification status, approval status)
- **View user details** (Telegram ID, email, alerts, schedules, verification status, approval status)
- **User approval workflow** (approve/reject users who have verified their email)
- **Edit user info** (reset email, manually verify, deactivate/reactivate)
- **Delete user** (with confirmation)
- **Export users** (CSV, XLSX, JSON)

### 3. **Alerts Management**
- **View all alerts** (filter by user, ticker, status)
- **Edit, pause, resume, or delete alerts**
- **Add alert for user** (manual override)
- **View alert history/logs**

### 4. **Schedules Management**
- **View all scheduled reports** (filter by user, ticker, time)
- **Edit, pause, resume, or delete schedules**
- **Add schedule for user**
- **View schedule history/logs**

### 5. **Broadcast & Messaging**
- **Send broadcast message** (to all users or filtered segment)
- **Send direct message** (to specific user)
- **View broadcast/message history**

### 6. **Logs & Monitoring**
- **View logs** (user commands, API errors, email delivery, admin actions)
- **Filter/search logs**
- **Log retention management** (e.g., auto-delete after 30 days)
- **Download logs**

### 7. **Statistics & Analytics**
- **User statistics** (active users, registration trends, verification rates, approval rates)
- **Usage statistics** (reports generated, alerts triggered, emails sent)
- **API usage stats** (calls per provider, error rates)
- **Charts and visualizations**
- **Pending approvals** (real-time count of users waiting for approval)

### 8. **Settings & Limits**
- **Set global limits** (max alerts/schedules per user)
- **Configure data providers** (API keys, priority order)
- **Manage admin accounts** (add/remove admins, set roles)
- **Feature toggles** (enable/disable modules, e.g., feedback)

### 9. **Feedback & Feature Requests**
- **View user feedback and feature requests**
- **Reply to feedback**

### 10. **Security & Audit**
- **Audit trail** (all admin actions logged)
- **Manual user verification**
- **Session and access logs**

## UI Mockups Description

### **A. Dashboard (Home)**
- **Top bar:** Quick stats (active users, reports today, alerts triggered, emails sent)
- **Sidebar:** Navigation menu (Users, Alerts, Schedules, Broadcast, Logs, Settings)
- **Main area:** Charts (user growth, activity heatmap), recent actions, quick links

### **B. Users Page**
- **Table:** List of users with columns (Telegram ID, Email, Verified, Approved, Alerts, Schedules, Status, Actions)
- **Actions:** View, Edit, Delete, Reset Email, Manually Verify, Approve/Reject
- **Search/filter:** By email, status, registration date, approval status

### **C. User Details Modal/Page**
- **Profile info:** Telegram ID, email, registration/verification dates, status
- **Tabs:** Alerts, Schedules, Logs, Feedback
- **Actions:** Edit, reset email, verify, deactivate

### **D. Alerts Management**
- **Table:** All alerts (User, Ticker, Price, Condition, Status, Actions)
- **Actions:** Pause, Resume, Edit, Delete
- **Filter:** By user, ticker, status

### **E. Schedules Management**
- **Table:** All schedules (User, Ticker, Time, Status, Actions)
- **Actions:** Pause, Resume, Edit, Delete
- **Filter:** By user, ticker, time

### **F. Broadcast/Message**
- **Form:** Compose message (text, optional image/file), select recipients (all/segment/user)
- **History:** List of previous broadcasts/messages, delivery stats

### **G. Logs & Analytics**
- **Tabs:** User Commands, API Errors, Email Delivery, Admin Actions
- **Table:** Timestamp, Type, User/Admin, Details
- **Charts:** API usage, error rates, user activity

### **H. Settings**
- **Global limits:** Set max alerts/schedules per user
- **API keys:** Manage provider keys and priorities
- **Admins:** List/add/remove admins, set roles

## Implemented Admin Panel Features

### Current Implementation Status
✅ **Fully Implemented and Functional**

### Access Information
- **URL:** `http://localhost:5000` (when running `admin_panel.py`)
- **Authentication:** Username/password via environment variables (`WEBGUI_LOGIN`, `WEBGUI_PASSWORD`)
- **Framework:** Flask-based web application

### Implemented Features

#### ✅ Dashboard
- **Statistics Cards:** Total users, verified users, approved users, pending approvals, active alerts, active schedules, open feedback
- **Pending Approvals Section:** Quick view of users waiting for approval with direct approve/reject buttons
- **Real-time Updates:** Statistics update automatically when accessed

#### ✅ User Management
- **User List:** Complete table with Telegram ID, email, verification status, approval status, language, admin status, limits
- **Approval Workflow:** 
  - Approve button for verified users who aren't approved
  - Reject button for verified users who aren't approved
  - Revoke button for already approved users
- **Manual Actions:** Verify users, reset emails, manage user limits

#### ✅ System Management
- **Alerts Management:** View, edit, pause, resume, delete all user alerts
- **Schedules Management:** View, edit, pause, resume, delete all scheduled reports
- **Broadcast Messaging:** Send messages to all users
- **Feedback Management:** View and respond to user feedback

#### ✅ Security Features
- **Login Protection:** All routes protected with `@login_required` decorator
- **Session Management:** Secure login/logout with Flask sessions
- **Audit Trail:** All admin actions logged with timestamps
- **Error Handling:** Proper validation and user feedback

### Usage Instructions
1. **Start the admin panel:** `python src/frontend/telegram/screener/admin_panel.py`
2. **Access the interface:** Open `http://localhost:5000` in your browser
3. **Login:** Use admin credentials set in environment variables
4. **Navigate:** Use the top navigation bar to access different sections
5. **Manage users:** Go to "Users" page to see approval buttons and user management options
6. **Monitor system:** Use "Dashboard" to see pending approvals and system statistics

**For a modern look and feel, consider using frameworks like Flask-Admin, Django Admin, or React-based dashboards.**
