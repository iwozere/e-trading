## Phase 1 — Critical Bug Fixes ✅

### 1.1 Triple-barrier label leakage — P07 evaluator

**File:** `src/ml/pipeline/p07_combined/evaluator.py`

`run_evaluation()` now splits raw OHLCV **before** computing any labels. A `tpl_bars`-wide buffer is left between each boundary to guarantee no label looks ahead into the adjacent segment.

Split ratios: **60% train / 20% val / 20% test** (buffers come out of the train and val ends).

Result dict keys added:
- `pf_val` — validation portfolio (Optuna target)
- `pf_test` — true OOS portfolio (artifacts only)
- `"pf"` — legacy alias for `pf_test` (backward compat)

### 1.2 Triple-barrier leakage — P08 evaluator

**File:** `src/ml/pipeline/p08_mtf/evaluator.py`

`P08Evaluator` now only overrides `prepare_data()` and inherits the fixed `run_evaluation()` from `P07Evaluator`. Duplicate override removed.

### 1.3 Optuna contamination — P07 optimize

**File:** `src/ml/pipeline/p07_combined/optimize.py`

- Objective scores on `res["pf_val"]`, never `res["pf_test"]`.
- Minimum trades threshold lowered 30 → 20.
- Guard added: `np.log10(max(total_trades, 2))` prevents log(0).

### 1.4 Optuna contamination — P08 optimize

**File:** `src/ml/pipeline/p08_mtf/optimize.py`

Same fix: `pf_val`, min-trades 10, `max(..., 2)` guard.

### 1.5 Blocking `input()` — P01 pipeline runner

**File:** `src/ml/pipeline/p01_hmm_lstm/run_pipeline.py`

- Added `--interactive` CLI flag (default `False`).
- Both `input()` calls are now guarded by `if is_interactive:`.

- Warning emitted when `--skip-stages 1` or `2` is passed.

### 1.6 Indicator-optimization contamination — P01

**File:** `src/ml/pipeline/p01_hmm_lstm/x_04_optuna_indicators.py`

Stage 4 now uses only the **first 60%** of OHLCV for indicator Optuna. The remaining 40% is untouched until Stage 5 (LSTM Optuna) and Stage 7 (final test).

## Phase 6 — Validation Infrastructure ✅

### 6.1 Walk-Forward Efficiency (WFE) gate

**File:** `src/ml/pipeline/p07_combined/pipeline.py` → `save_artifacts()`

After computing the in-sample Sharpe ratio, runs `P07RobustnessChecker.run_all_checks()` and computes:

```
wfe = avg_oos_sharpe / is_sharpe
```

If `wfe < 0.5`: logs a warning and returns `False` (does not write `completed.flag`).  
If the WFE check raises any exception: logs a warning and allows the run to proceed.

### 6.2 TRF cache TTL validation

**File:** `src/ml/pipeline/shared/trf_downloader.py`

Added `_is_cache_fresh(path: Path, max_age_days: int = 1) -> bool`:
```python
age_days = (datetime.now().timestamp() - path.stat().st_mtime) / 86400.0
return age_days <= max_age_days
```

`download_trf()` now checks `_is_cache_fresh(output_file)` before returning the cached result. If stale, forces re-download and logs `"TRF cache is stale for data_date=… — forcing refresh"`.

`get_trf_correction_factor()` now logs data_date, total_volume, and short_volume alongside every correction factor at `_logger.debug()` level.

All f-string log calls converted to lazy `%s` format.

### 6.3 Rolling memory bootstrap health check

**File:** `src/ml/pipeline/p06_emps2/rolling_memory.py`

Added to `RollingMemoryScanner`:
- `_state_path: Path` property — `results_base_path / ".rolling_memory_state.json"`
- `_load_state() -> dict` — reads JSON state file; returns `{}` on missing or corrupt file.
- `_save_state(state: dict)` — writes JSON state; logs warning on failure (never raises).
- `check_bootstrap_health(phase1_count: int, lookback_days: Optional[int] = None)`:
  - Persists `first_run_date`, `last_run_date`, `cumulative_phase1_count`.
  - Emits `_logger.error()` when `active_days > lookback_days` and `cumulative_phase1_count == 0`.

### 6.4 `shuffle=True` audit

Audited all `shuffle=True` occurrences across the codebase. All four instances are in PyTorch `DataLoader` for mini-batch shuffling **within already-split training data**. All `train_test_split` calls use `shuffle=False`. No fixes required.

## Phase 7 — Reclassifications & Minor Fixes ✅

### 7.1 P13 reclassified as shared risk overlay

**File created:** `src/ml/pipeline/shared/risk_overlay/vix_position_sizer.py`

`VixPositionSizer` — pipeline-agnostic VIX-based position sizer extracted from P13 (`p13_bdsh`):
- `compute_z_scores(vix_series)` — rolling Z-scores.
- `compute_exposures(vix_series)` — lagged Z-scores mapped through configurable tier ladder to target exposures in [0.0, 1.0]. Lag of 1 bar prevents look-ahead.
- `scale_position_size(base_size, vix_series, as_of_date)` — convenience method for on-the-fly scaling.

`VixPositionSizerConfig` dataclass with `z_lookback`, `exit_z_threshold`, and `entry_tiers` (list of `{z_threshold, allocation}` dicts).

### 7.2 P09 walk-forward cointegration backtest mode

**File:** `src/ml/pipeline/p09_arbitrage/cointegration_analyzer.py`

Added `walk_forward_backtest(series_a, series_b, n_splits=5, entry_z=2.0, exit_z=0.5) -> Dict`:
- Uses `TimeSeriesSplit`; estimates hedge ratio on train window only (OOS-safe).
- Simulates mean-reversion strategy (long/short spread, entry/exit Z-score gates) on held-out test window.
- Returns `{fold_metrics, avg_sharpe, avg_total_return, avg_half_life, n_folds}`.

Added `from sklearn.model_selection import TimeSeriesSplit` and extended `typing` imports.

### 7.3 P18 dedup fix

**File:** `src/ml/pipeline/p18_institutional_flow_tracker/pipeline.py` → `_collect_watchlist_tickers()`

Replaced O(n²) seen-list loop:
```python
# Before
seen: List[str] = []
for t in tickers:
    if t and t not in seen:
        seen.append(t)
return seen[:max_tickers]

# After
return list(dict.fromkeys(t for t in tickers if t))[:max_tickers]
```
