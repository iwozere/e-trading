# Pipeline Specification: P12 Order Flow (Market Microstructure)

## Overview
This document specifies the architecture and implementation details for the `p12_order_flow` pipeline. Unlike traditional OHLCV-based strategies, this pipeline relies exclusively on crypto-native market microstructure and order flow data. The primary objective is to build predictive models that capitalize on structural imbalances, extreme liquidations, funding rate anomalies, and shifts in Open Interest (OI) that precede major price movements.

## Architecture & Integration Strategy
The `p12_order_flow` pipeline will ingest alternative data sources directly relevant to derivatives markets (perpetual futures). The core of this pipeline is dependent on accurate historical and live data collection, which will eventually be integrated into the central `binance_data_downloader.py`.

### Output Destination
All execution outputs, models, and state files will be persisted in `results/p12_order_flow/YYYY-MM-DD/`.

## Component Design

### 1. Data Acquisition Strategy
Acquiring accurate order flow data is the foundational requirement for this pipeline. The following data sources and integration paths will be utilized:

#### A. Binance Futures API (Free / Direct Integration)
This is the primary target for integration into `binance_data_downloader.py`.
- **Funding Rates**: Fetch historical and live funding rates (`GET /fapi/v1/fundingRate`). Useful for detecting directional bias and over-leveraged sentiment.
- **Open Interest (OI)**: Retrieve historical OI (`GET /futures/data/openInterestHist`) to measure the total number of outstanding derivative contracts. Rising OI with rising price indicates trend continuation; rising OI with flat price often precedes volatility.
- **Long/Short Ratio**: Utilize the Global Long/Short Ratio (`GET /futures/data/globalLongShortAccountRatio`) as a baseline for retail/institutional positioning.

#### B. Coinglass API (Freemium / Premium Aggregation)
If Binance-only data proves insufficient, Coinglass provides industry-standard, aggregated order flow data.
- **Cumulative Volume Delta (CVD)**: Essential for determining whether market market-buy or market-sell orders are dominating the tape.
- **Aggregated Liquidations**: Clean historical data on liquidations across all major exchanges. High liquidations often signal a local top/bottom.

#### C. Binance Vision (Raw Historical Data Dumps)
For deep, tick-level backtesting across years of data without rate limits.
- Download raw CSV dumps of trades, liquidations, and order book snapshots directly from Binance's historical data portal.

### 2. `config.py`
Configuration defining the order flow thresholds and lookback periods.
**Key Parameters**:
- `extreme_funding_threshold`: 0.05% (per 8 hours)
- `liquidation_spike_zscore`: 3.0 (Identifies flush events)
- `oi_growth_threshold`: 10% increase over 24h
- `cvd_divergence_window`: 15 (Periods for checking price/CVD divergence)

### 3. `order_flow_pipeline.py`
The orchestration engine.
- **Stage A (Data Ingestion)**: Load OHLCV data merged with Funding, OI, Liquidations, and CVD.
- **Stage B (Feature Engineering)**: Compute derived metrics, such as:
  - `OI_Normalized`: (Current OI - Rolling Min OI) / (Rolling Max OI - Rolling Min OI)
  - `Funding_Regime`: Moving average of the funding rate to determine structural bias.
  - `Price_CVD_Divergence`: Boolean flag if price makes a higher high but CVD makes a lower high.
- **Stage C (Model Training/Inference)**: Train a classifier (e.g., XGBoost, LightGBM) to predict the probability of a liquidation cascade or sharp mean reversion based on the structural imbalances.
- **Stage D (Signal Generation)**: Issue execution signals when probabilities cross the confidence threshold.

### 4. `microstructure_analyzer.py`
The dedicated module for detecting specific order flow setups.
#### Scenarios Detected:
- **The Liquidation Flush**: Price drops sharply, triggering massive long liquidations (Z-score > 3), while OI drops correspondingly. This often outputs a high-probability mean-reversion (LONG) signal.
- **The Squeeze Divergence**: Price is stagnant or slowly rising, but the Long/Short ratio is heavily skewed short and funding is deeply negative. This outputs a short-squeeze (LONG) signal.

## Execution Outputs
- `pipeline.log`: Full execution trace.
- `01_raw_microstructure_data.csv`: The merged dataset from Binance/Coinglass.
- `02_engineered_order_flow_features.csv`: Processed features ready for ML ingestion.
- `03_imbalance_alerts.csv`: Real-time alerts for extreme funding or OI conditions.
- `04_predictive_signals.csv`: Actionable trading signals generated by the order flow model.
