# Architect Review: Prioritized Improvements
**Date:** 2026-04-30  
**Scope:** Full codebase audit ‚Äî `src/` (845 Python files across 18 modules)

---

## P0 ‚Äî Critical / Quick Wins (do these now)

### 1. Trading module: near-zero test coverage
**Why critical:** 63 production files, 1 test. `StrategyManager` (sole config loader), `StrategyHandler`, `StrategyInstance`, all broker integrations (Binance, IBKR), and the risk controller are completely unverified.  
**Quick win path:** Start with `MockBroker` + `strategy_registry.py` + `config_validator.py` ‚Äî these have no external dependencies and cover the critical startup path.  
**Files:** `src/trading/strategy_manager.py`, `src/trading/broker/mock_broker.py`, `src/trading/config_validator.py`, `src/trading/risk/`

### 2. Config system: 0 tests, 0 docs
**Why critical:** `config/` is the single-point-of-failure for every module. Broken config loading silently misfires strategies, wrong broker endpoints, or corrupts position sizing.  
**Quick win path:** Unit-test `config_loader.py` (YAML/JSON parsing), `config_manager.py` (singleton behaviour), and `configuration_factory.py` (manifest hydration). Add a `docs/` folder with at least `README.md`.  
**Files:** `src/config/config_loader.py`, `src/config/config_manager.py`, `src/config/configuration_factory.py`

### 3. Error handling: resilience patterns untested
**Why critical:** Circuit breaker and retry logic protect live trading from cascading failures. Untested circuit breakers are false security ‚Äî they may flip at the wrong threshold or never reset.  
**Quick win path:** Parameterised unit tests for `CircuitBreaker` (open/half-open/closed states), `RetryManager` (backoff intervals, max attempts), `ResilienceDecorator` (exception propagation).  
**Files:** `src/error_handling/circuit_breaker.py`, `src/error_handling/retry_manager.py`, `src/error_handling/resilience_decorator.py`

### 4. Portfolio / P&L alerts: 17 files, 0 tests
**Why critical:** Financial calculations (P&L, drawdown thresholds) produce real money alerts. Off-by-one errors or wrong sign conventions cause false alerts or missed alerts.  
**Quick win path:** Test the core alert evaluation logic in `pnl_alert/` with fixed fixtures covering edge cases (negative P&L, zero position, boundary thresholds).  
**Files:** `src/portfolio/pnl_alert/`

## P1 ‚Äî High Impact (next sprint)

### 5. Split `data_manager.py` (3,063 lines)
**Problem:** One file is a facade over 13+ providers, unified cache, OHLCV normalisation, provider failover, and rate limiting. It is the hardest file in the project to test and change safely.  
**Recommended split:**
- `provider_selector.py` ‚Äî failover & priority logic
- `ohlcv_normaliser.py` ‚Äî contract enforcement (tz-naive index, column names)
- `cache_coordinator.py` ‚Äî unified cache orchestration
- `data_manager.py` ‚Äî thin facade importing the above  
**Files:** `src/data/data_manager.py`

### 6. ML pipelines: 188 files, 0 tests
**Problem:** 13 production ML pipelines (P00‚ÄìP14) with Optuna HPO, MLflow registry, and Backtrader integration have zero test coverage. Model regressions go undetected.  
**Recommended approach:** Don't test training scripts end-to-end. Instead write focused tests per pipeline for: data preprocessing transforms, feature engineering functions, signal generation logic, and model serialisation/deserialisation. Start with P05/P06/P10 (most actively used).  
**Files:** `src/ml/p05_*/`, `src/ml/p06_*/`, `src/ml/p10_*/`

### 7. Refactor `base_broker.py` (1,689 lines)
**Problem:** Abstract broker protocol mixes order management, position tracking, balance management, event handling, and notification dispatch in one class.  
**Recommended split:** Extract `OrderManager`, `PositionTracker`, `BalanceTracker` as composable mixins. `BaseBroker` becomes a thin coordinator.  
**Files:** `src/trading/broker/base_broker.py`

### 8. Scheduler service: 1,311-line service, 0 tests
**Problem:** APScheduler integration with zero test coverage. Missed schedules or duplicate job creation in production are silent.  
**Quick win:** Test `scheduler_service.py` with a mock APScheduler backend ‚Äî job creation, cancellation, and execution callbacks.  
**Files:** `src/scheduler/scheduler_service.py`

## P2 ‚Äî Important (next month)

### 9. API module: 1 doc for 28 files
**Problem:** FastAPI routes (`notification_routes.py` at 986 lines, `trading_bot_routes.py`, `jobs_routes.py`) have minimal route-level docstrings. The OpenAPI spec exposed to consumers is skeletal.  
**Fix:** Add `summary`, `description`, and `response_model` to every route decorator. Add `README.md` + `docs/` folder per CLAUDE.md convention.  
**Files:** `src/api/`

### 10. `util/` module: 11 files, 0 tests, 0 docs
**Problem:** Shared utilities used project-wide (`date_time_encoder.py`, `drawdowns.py`, `extract_tickers.py`, `tickers_list.py`) have no tests. Silently broken helpers propagate errors everywhere.  
**Fix:** Unit tests for each utility + `README.md`.  
**Files:** `src/util/`

### 11. `screeners/` module: 0 tests
**Problem:** Stock/crypto screening logic (IBKR integration, discovery, filtering) is entirely untested. Wrong screening results silently feed bad tickers to strategies.  
**Files:** `src/screeners/`

### 12. Add `pyproject.toml` and consolidate `requirements*.txt`
**Problem:** Four separate requirements files (`requirements.txt`, `requirements-dev.txt`, `requirements-test.txt`, `requirements-webui.txt`) without a `pyproject.toml`. The project is not pip-installable, and dependency resolution is manual.  
**Fix:** Introduce `pyproject.toml` (PEP 517/518) with optional dependency groups (`[dev]`, `[test]`, `[webui]`). Retire the four separate files.

### 13. Fix import path fragility in `api/main.py`
**Problem:** `api/main.py` uses complex `sys.path` manipulation to avoid a name collision with the stdlib `config` module. This is fragile and violates CLAUDE.md pathlib convention.  
**Fix:** Rename `src/config/` to `src/app_config/` or use `importlib` namespace packages. Update all imports.  
**Files:** `src/api/main.py`, `src/config/`

## P3 ‚Äî Nice to Have (backlog)

### 14. `common/alerts/alert_evaluator.py` (1,696 lines) ‚Äî extract sub-evaluators
One class handling all alert types (price, volume, P&L, sentiment, fundamentals). Extract per-domain evaluators under `common/alerts/evaluators/`.

### 15. `indicators/service.py` (1,298 lines) ‚Äî split into domain groups
Technical indicators service mixing TA-Lib adapters, custom indicators, and configuration. Extract into `ta_lib_service.py`, `custom_indicator_service.py`, `indicator_config_service.py`.

### 16. `file_based_cache.py` (1,541 lines) ‚Äî extract Parquet I/O layer
Cache management, rollover logic, and Parquet serialisation are interleaved. Extract a `ParquetStore` I/O class for independent testing.

### 17. Improve `docker-compose.yml`
Current Docker setup is minimal. Add health checks, volume mounts for `/data` and `/logs`, environment variable injection from `.env`, and a `dev` profile that mounts source code for live reload.

### 18. Enforce 80% coverage target project-wide
`pytest.ini` sets 80% coverage but only enforces it on `src/data`. Extend the `--cov-fail-under=80` requirement to `src/trading`, `src/config`, and `src/error_handling` as tests are added.
